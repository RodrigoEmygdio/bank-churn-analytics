# ADR Review Checklist

## Decision Quality

- [ ] Is every architectural decision justified?

- [ ] Is every decision backed by evidence?

- [ ] Are alternatives compared fairly?

---

## Architecture

- [ ] Does the ADR describe responsibilities?

- [ ] Does it avoid implementation details?

- [ ] Can folders change without invalidating the ADR?

---

## Traceability

- [ ] Is the ML report referenced?

- [ ] Are notebooks reflected?

- [ ] Are approved discussions preserved?

---

## Longevity

- [ ] Will this ADR still make sense after a refactor?

---

## Consistency

- [ ] Does it contradict previous ADRs?

- [ ] Does it preserve approved architecture?
