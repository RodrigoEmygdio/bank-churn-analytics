# ADR Self Review

Before returning an ADR, verify:

## Scope

Is this architecture?

Or implementation?

---

## Decisions

Does every decision explain WHY?

---

## Alternatives

Were alternatives compared objectively?

---

## Responsibilities

Did I describe responsibilities instead of files?

---

## Evidence

Can every architectural decision be traced back to evidence?

---

## Longevity

Will this ADR remain valid if the implementation changes?

---

## Independence

Does the ADR avoid prescribing implementation?

---

## Final Question

If the code were rewritten tomorrow, would this ADR still be correct?

If not, rewrite the affected sections.
