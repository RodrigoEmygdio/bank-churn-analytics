# ADR Anti-Patterns

## Anti-Pattern

Decision:

Use src/services.

Problem:

Implementation detail.

Better:

Separate prediction from orchestration.

---

## Anti-Pattern

Use pytest.

Problem:

CI detail.

Better:

Implementation must be objectively verifiable.

---

## Anti-Pattern

Rejected because AGENTS.md says so.

Problem:

Circular reasoning.

Better:

Rejected because it contradicts the analytical conclusions obtained during model evaluation.

---

## Anti-Pattern

Describe classes.

Problem:

Implementation.

Better:

Describe architectural responsibilities.

---

## Anti-Pattern

Mix architecture with testing.

Problem:

Multiple document responsibilities.

Better:

Reference testing requirements without defining test implementation.
